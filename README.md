# Sensor Time-Series Pipeline — Group 12

End-to-end time-series pipeline: EDA & forecasting model (Task 1), relational
+ NoSQL database design (Task 2), CRUD/time-series API (Task 3), and a
consolidated prediction script (Task 4).

**Dataset:** [Temperature, Humidity, Pressure & Illuminance Sensor Dataset](https://www.kaggle.com/datasets/patrickfleith/temperature-humidity-pressure-illuminance) (Kaggle)

## Setup

```bash
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
```

### 1. Run the notebook
Open `notebook/task1_timeseries_pipeline.ipynb` in Colab or Jupyter, run top
to bottom. It downloads the dataset via `kagglehub`, does EDA/preprocessing,
trains and tunes two model families, and saves everything needed for Task 4
into `notebook/artifacts/`.

### 2. Load the databases
```bash
mysql -u root -p < sql/schema.sql
python mongodb/mongodb_design.py   # seeds MongoDB with sample documents
```

### 3. Run the API
```bash
export MYSQL_URL="mysql+pymysql://root:password@localhost:3306/sensor_timeseries"
export MONGO_URL="mongodb://localhost:27017/"
uvicorn api.app:app --reload
```
Interactive docs at `http://localhost:8000/docs`.

### 4. Run a prediction
```bash
cd scripts
python predict.py --sensor-id 1 --api-base http://localhost:8000 --model rf
```
